# 飞机大战项目开发文档

## 1. 项目概览

这是一个基于 Pygame 的纵版飞行射击游戏。当前有效主程序位于 `code/test.py`，玩家控制英雄飞机移动，系统自动发射子弹，敌机会按分数动态生成。击毁敌机后掉落宝石和道具，宝石负责加分，道具负责回血、护盾、升级火力和追踪弹。分数达到指定阈值后会进入 Boss 警告和 Boss 战。

项目目前没有独立的包管理文件，运行依赖主要是 Python 和 Pygame。素材资源集中在 `code/assets/images` 和 `code/assets/sounds`。

## 3. 运行方式

进入 `code` 目录运行主程序：

```bash
cd code
python test.py
```

运行前需要安装 Pygame：

```bash
pip install pygame
```

注意事项：

- 当前代码使用 Windows 字体路径 `C:/Windows/Fonts/simhei.ttf`，在非 Windows 环境需要调整字体加载。
- 当前菜单音乐和开始音效存在硬编码绝对路径 `C:\Users\bill\Desktop\期末实训\assets\sounds\...`，在其他电脑上可能无法播放，需要改成从 `resource_manager` 或 `settings.SOUNDS_DIR` 读取。
- Python 文件已通过 `py_compile` 语法检查，但这不等于游戏运行时完全无错误。

## 4. 入口与主循环

入口文件是 `code/test.py`。

程序启动流程：

1. `pygame.init()` 初始化 Pygame。
2. 创建 `Game` 对象。
3. `Game.run()` 创建窗口、加载全部资源、创建英雄精灵。
4. 进入无限循环：
   - 处理退出、鼠标点击、暂停、开始、重开等事件。
   - 根据游戏状态更新英雄、敌机、子弹、道具、Boss。
   - 自动射击。
   - 生成敌机或触发 Boss。
   - 做碰撞检测。
   - 清理失效精灵。
   - 绘制背景、精灵、UI、血条、分数、暂停菜单等。
   - `pygame.display.flip()` 刷新画面。
   - `clock.tick(FPS)` 控制帧率。

核心状态由 `GameState` 表示：

```text
START        开始界面
PLAYING      普通游戏中
PAUSED       暂停
GAME_OVER    游戏结束
BOSS_WARNING Boss 出场警告
BOSS_BATTLE  Boss 战
```

当前代码里 `PAUSED` 被定义了两次，后一次值为 `6`，会覆盖前一次值 `2`。这不影响“是否等于 PAUSED”的判断，但会让枚举值不直观。

## 5. 主要模块逻辑

### 5.1 `settings.py`

负责全局配置：

- `SCREEN_WIDTH = 480`
- `SCREEN_HEIGHT = 700`
- `FPS = 60`
- `CURRENT_DIR` 当前 `code` 目录
- `IMAGES_DIR` 图片目录
- `SOUNDS_DIR` 声音目录

后续新增资源时应优先从这里派生路径，避免写绝对路径。

### 5.2 `resource_manager.py`

`ResourceManager` 是一个单例资源管理器，暴露全局对象 `resource_manager`。

主要职责：

- `load_all_resources()` 一次性加载图片、音效、字体。
- `load_image()` 加载图片，并可按固定尺寸缩放。
- `load_sound()` 加载音效。
- `load_font()` 加载字体。
- `get_image(key)`、`get_sound(key)`、`get_font(key)` 根据 key 取缓存资源。
- `play_sound(key)` 播放指定音效。

资源 key 示例：

- 英雄：`hero`、`damaged`、`death`
- 玩家子弹：`bullet`
- 敌机：`enemy1`、`enemy2`、`enemy3_n`
- 敌机死亡：`enemy1_down`、`enemy2_down`、`enemy3_down`
- Boss：`boss_idle`、`boss_death`、`boss_cloak`
- 道具：`item_heal`、`item_shield`、`item_upgrade`、`item_track`
- 宝石：`gem1`、`gem2`、`gem3`、`gem4`
- 音效：`background`、`bullet`、`bomb`、`bgm_boss`、`warning` 等

### 5.3 `sprites/Hero.py`

`Hero` 是玩家飞机精灵。

主要属性：

- `rect`：位置和碰撞区域。
- `speed = 5`：移动速度。
- `max_hp = 10`、`live = 10`：生命值。
- `active`：是否仍在游戏中。
- `invincible`、`invincible_timer`：受伤后的短暂无敌。
- `is_dying`：死亡动画状态。

主要行为：

- `move(direction)`：根据方向移动，并限制在屏幕范围内。
- `update()`：播放待机、受伤闪烁或死亡动画。
- `hit()`：被击中时扣 1 点生命并进入无敌状态。
- `heal(amount=3)`：回血，不超过最大生命值。
- `hp_ratio`：返回血条比例。

输入控制在 `Game.hero_control()` 中完成，支持方向键和 WASD。

### 5.4 `sprites/Bullet.py`

这是当前实际使用的玩家子弹类。

构造参数：

- `x, y`：初始位置。
- `speed`：速度。
- `vx`：横向速度，用于扇形散射。
- `image_key`：资源 key，默认 `bullet`。
- `img_idx`：子弹图片序号。
- `target`：追踪目标。

更新逻辑：

- 有目标且目标仍 active 时，按目标方向移动。
- 无目标时向上移动，并叠加横向速度。
- 超出屏幕后设为 inactive 并 `kill()`。

`code/Bullet.py` 是旧版或残留文件，里面读取的是 `res.get_image('bullet.png')`，但资源管理器实际 key 是 `bullet`，当前主流程没有使用它。

### 5.5 `sprites/Enemy.py`

`Enemy` 表示普通敌机。敌机参数集中在 `ENEMY_TYPES`：

```text
1 型敌机：HP 1，速度 1-3，分值 1
2 型敌机：HP 3，速度 1-2，分值 3
3 型敌机：HP 6，速度 1-2，分值 5
```

主要逻辑：

- 正常状态下向下移动。
- 超出屏幕后失效。
- 被击中时扣血。
- 血量归零后进入死亡动画，动画结束后失效。
- 非致死伤害时可短暂显示受伤图。

当前设计里普通敌机的直接分值主要没有用于玩家子弹击杀加分，玩家得分主要来自击杀后掉落的宝石。

### 5.6 `sprites/EnemyManager.py`

`EnemyManager` 负责普通敌机生成、难度增长和 Boss 触发。

分数分层：

```text
0: score < 200
1: score < 500
2: score < 1000
3: score < 2000
4: score < 5000
5: score >= 5000
```

难度随分数提高而变化：

- 敌机生成间隔缩短。
- 高级敌机权重提高。
- 敌机速度倍率提高。
- 2000 分后一次可生成 2 只敌机。
- 道具掉落率随分数升高而降低。

Boss 触发：

- 初始 Boss 阈值为 `40`。
- 达到阈值后 `update()` 返回 `"WARNING"`。
- 主循环进入 `BOSS_WARNING`，倒计时结束后调用 `spawn_boss()`。
- Boss 被击败后阈值增加 `1000`。

当前初始 Boss 阈值是 40 分，比较低，可能是调试用数值。

### 5.7 `sprites/Boss.py`

包含两个类：

- `BossBullet`：Boss 子弹。
- `Boss`：Boss 本体。

Boss 出场：

- 初始位置在屏幕上方。
- 通过 `_enter_screen()` 慢慢移动到 `top = 80`。
- 到位后 `arrived = True`，开始可被有效攻击。

Boss 行为：

- 横向缓慢追踪英雄。
- 左右漂移。
- 播放待机动画。
- 每 10 秒触发隐形，隐形结束后瞬移。
- 血量低于一半时进入二阶段。
- 二阶段切换时有 5 秒无敌和屏幕震动。

Boss 攻击：

- 追踪射击：周期性向玩家方向发射 3 发。
- 扇形散射：周期性发射 9 发扇形弹幕。
- 二阶段螺旋弹幕：分数达到 500 后解锁。
- 二阶段追踪爆裂弹：分数达到 1000 后解锁。

Boss 死亡：

- `hit()` 中血量归零后进入死亡动画。
- 死亡动画结束后 `active = False`。
- 主循环检测到 Boss inactive 后调用 `end_boss_battle()`。

### 5.8 `sprites/Item.py`

`Item` 同时表示功能道具和宝石。

功能道具：

```text
heal     回血，权重 35
shield   护盾/无敌，权重 15
upgrade  升级火力，权重 35
track    追踪弹，权重 15
```

宝石配置：

```text
gem1：1 分
gem2：3 分
gem3：5 分
gem4：10 分
```

掉落逻辑：

- 敌机被玩家子弹击中时，按敌机类型调用 `Item.spawn_gems()` 掉落宝石。
- 同时根据 `EnemyManager.get_item_drop_rate(score)` 决定是否掉落功能道具。
- 宝石会在英雄 120 像素范围内自动吸附。

拾取效果：

- 宝石：增加分数并播放 `crystal`。
- 回血：恢复 3 点生命。
- 护盾：进入无敌状态。
- 升级：提高 `game.upgrade_level`，满级后触发暴走。
- 追踪：设置 `game.track_timer`。

## 6. 游戏状态流

```text
START
  └─ 点击非喇叭区域
      └─ 播放开始动画
          └─ PLAYING

PLAYING
  ├─ 点击暂停按钮 -> PAUSED
  ├─ 英雄死亡 -> GAME_OVER
  └─ 分数达到 Boss 阈值 -> BOSS_WARNING

BOSS_WARNING
  └─ 倒计时结束 -> BOSS_BATTLE

BOSS_BATTLE
  ├─ Boss 死亡 -> PLAYING
  ├─ 英雄死亡 -> GAME_OVER
  └─ 点击暂停按钮 -> PAUSED

PAUSED
  ├─ 返回游戏 -> PLAYING 或 BOSS_BATTLE
  ├─ 重新开始 -> PLAYING
  └─ 返回主界面 -> START

GAME_OVER
  ├─ 点击返回主界面按钮 -> START
  └─ 点击其他区域 -> PLAYING
```

## 7. 射击与碰撞逻辑

### 玩家自动射击

`Game.auto_shoot()` 根据状态和道具计时器决定射击方式：

- 普通状态：每 35 帧发射。
- 追踪状态：每 25 帧发射，速度更快，使用第 4 张子弹图。
- 暴走状态：每 15 帧发射 5 发扇形子弹。
- 升级等级控制弹道数量，最多 3 路。

追踪目标选择逻辑是从当前敌机组里找一个 active 且不在 dying 状态的敌人。

### 碰撞检测

`Game.check_collision()` 负责：

- 玩家子弹和敌人/Boss 碰撞。
- 英雄和普通敌机碰撞。
- Boss 子弹和英雄碰撞。
- 英雄拾取道具和宝石。

当前碰撞使用：

- 子弹与敌人：`pygame.sprite.groupcollide(..., False, False)`
- 英雄与敌人：`pygame.sprite.collide_mask(...)`
- 英雄与道具：`pygame.sprite.spritecollide(...)`

## 8. 绘制逻辑

主循环内的绘制顺序：

1. 计算 Boss 震屏偏移。
2. 绘制滚动背景 `bg_scroll` 两张拼接。
3. 绘制 `all_sprites_group` 中全部精灵。
4. 根据状态绘制开始页、结束页、Boss 警告、暂停菜单。
5. 绘制右上角喇叭和暂停按钮。
6. Boss 战时绘制 Boss 血条。
7. 游戏中绘制分数和英雄生命条。

背景滚动通过 `bg_offset = (bg_offset + 1) % 1130` 实现。

## 9. 音频逻辑

音效资源大部分由 `resource_manager` 管理：

- 普通背景音乐：`background`
- Boss 音乐：`bgm_boss`
- 子弹：`bullet`
- 爆炸：`bomb`、`boom_l`
- 道具：`life`、`shield`、`upgrade`、`rush`
- 宝石：`crystal`
- 警告：`warning`

当前存在两处硬编码绝对路径：

- `_play_menu_bgm()` 中的 `menu_bgm.ogg`
- 开始界面点击后播放的 `yahou.wav`

建议统一改成 `res.load_all_resources()` 里注册 key，再通过 `res.get_sound()` 或 `res.play_sound()` 播放。

## 10. 当前已知问题和维护风险

1. `GameState.PAUSED` 重复定义，后者覆盖前者。
2. `test.py` 同时承担主循环、UI、状态机、碰撞、音频控制，文件职责偏重。
3. 菜单音乐和开始音效使用了作者本机绝对路径，移植后可能报错。
4. `code/Bullet.py` 是旧版残留，逻辑与 `sprites/Bullet.py` 不一致，容易误导后续开发。
5. `EnemyManager._boss_threshold` 初始值为 40，Boss 很早出现，可能不是正式数值。
6. `EnemyManager.get_bullet_damage()` 目前没有接入玩家子弹伤害，Boss 和敌机实际仍按 `enemy.hit()` 默认 1 点伤害处理。
7. `Boss.shoot()` 里创建了 `_is_burst` 的大子弹，但当前没有实现爆裂后的二次分裂逻辑。
8. `BossBullet` 计算了 `bullet_damage`，但碰撞英雄时没有使用这个伤害值，英雄每次仍只扣 1 点。
9. 玩家子弹击杀普通敌机时没有直接把 `Enemy.hit()` 返回分数加到 `game.score`，当前主要靠宝石加分。
10. 资源名包含中文文件名，跨平台和压缩发布时需要注意编码。
11. 项目中有 `__pycache__` 和 `.idea`，如果要提交代码，通常不建议纳入版本库。
12. `assets/new` 是 4399 H5 页面/脚本和 `ffmpeg.exe`，当前 Python 游戏没有引用，建议标注来源或移出主工程。

## 11. 后续开发建议

### 拆分 `test.py`

建议逐步拆分为：

```text
main.py                 # 程序入口
game.py                 # Game 主类和主循环
game_state.py           # GameState
ui.py                   # 开始页、暂停页、血条、文本绘制
collision.py            # 碰撞处理
audio.py                # 音乐切换与音效播放
```

先做小步迁移，确保每次拆分后都能运行，不建议一次性大改。

### 统一资源管理

所有图片、音效、字体都应通过 `resource_manager.py` 注册和读取。新增资源时建议：

1. 把文件放入 `assets/images` 或 `assets/sounds`。
2. 在 `load_all_resources()` 添加明确 key。
3. 业务代码只通过 key 访问，不直接写文件路径。

### 数值配置外置

敌机、Boss、道具概率、分数阈值现在分散在多个 Python 文件里。后续可以集中到配置模块，例如：

```text
config/enemy_config.py
config/item_config.py
config/boss_config.py
```

这样调难度时不用改核心逻辑。

### 明确得分规则

当前注释写的是“得分全靠宝石”，但代码中敌人和 Boss 的 `hit()` 又返回分值。建议明确其中一种规则：

- 方案 A：只通过宝石得分，删除或忽略敌机直接分值。
- 方案 B：击杀敌机直接加分，宝石作为额外奖励。

选择后同步修改注释和碰撞逻辑。

### 增加基础测试或检查脚本

Pygame 项目不容易做完整自动化测试，但可以先做基础检查：

- Python 语法检查：`python -m py_compile ...`
- 资源完整性检查：遍历 `resource_manager.load_all_resources()` 中引用的文件是否存在。
- 关键类初始化检查：在无窗口或 dummy video driver 下验证资源加载。

```text
玩家飞机：code/sprites/Hero.py

敌机生成：code/sprites/EnemyManager.py
Boss：code/sprites/Boss.py
道具宝石：code/sprites/Item.py
```

常用调整点：

```text
改窗口大小：settings.py
改帧率：settings.py
改英雄血量/速度：sprites/Hero.py
改子弹速度/射击频率：test.py 的 auto_shoot()
改敌机血量/速度/分值：sprites/Enemy.py 的 ENEMY_TYPES
改敌机生成节奏：sprites/EnemyManager.py
改 Boss 血量/技能：sprites/Boss.py
改道具概率/宝石分值：sprites/Item.py
改图片音效资源：resource_manager.py
```
