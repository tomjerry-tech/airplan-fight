# 飞机大战公网部署文档

本文档用于把当前项目部署到阿里云 ECS，并通过宝塔面板管理后端服务、MySQL 和域名访问。

当前项目不是纯静态项目，不能只上传到静态网站托管。项目需要：

- Flask 后端：登录、房间、排行榜、健康检查接口
- Flask-SocketIO：双人实时对战同步
- MySQL：保存玩家、分数、房间历史
- Pygame 客户端：玩家电脑本地运行，通过公网地址连接后端

## 1. 当前结论

可以使用：

```text
阿里云域名 + 阿里云 ECS + 宝塔面板 + MySQL + Flask/SocketIO
```

但是要注意：

```text
服务器只部署后端和数据库。
玩家客户端仍然是本地 Pygame 程序，不是网页游戏。
```

公网双人联机的原理是：

```text
玩家 A 电脑 Pygame 客户端
        |
        | HTTP + WebSocket
        v
阿里云 ECS 上的 Flask/SocketIO 后端
        ^
        | HTTP + WebSocket
玩家 B 电脑 Pygame 客户端
```

只要两台电脑都能访问服务器公网地址，双人就不限制局域网。

## 3. 阿里云准备

### 3.1 购买 ECS

购买阿里云 ECS 后，记录：

```text
公网 IP：例如 1.2.3.4
系统账号：root 或普通用户
系统密码/密钥：你自己设置的登录凭据
```

### 3.2 安全组放行端口

在阿里云控制台找到 ECS 安全组，至少放行：

```text
22    SSH 远程登录
80    HTTP
443   HTTPS
8888  宝塔面板默认端口，安装后建议改掉
```

如果暂时不配置 Nginx 反向代理，也可以临时放行：

```text
5000  Flask 后端端口
```

正式使用时建议让 Nginx 监听 80/443，再反向代理到本机 5000，不直接暴露 5000。

### 3.3 域名解析

在阿里云 DNS 控制台添加 A 记录：

```text
主机记录：game
记录类型：A
记录值：你的 ECS 公网 IP
```

假设你的域名是：

```text
example.com
```

则后端域名可以设置为：

```text
game.example.com
```

解析完成后，本地可以测试：

```bat
ping game.example.com
```

能解析到 ECS 公网 IP 即可。

## 7. MySQL 配置

### 7.1 创建数据库

在宝塔 MySQL 管理里创建数据库：

```text
数据库名：airplane_game
用户名：airplane_game
密码：设置一个强密码
访问权限：本地服务器
```

不要在公网服务器上继续用：

```text

```

这个只适合本机开发。

### 7.2 初始化表

进入项目目录：

```bash
cd /www/wwwroot/airplan_game
```

设置数据库环境变量：

```bash
export AIRPLANE_DB_HOST=127.0.0.1

export AIRPLANE_DB_USER=airplane_game
export AIRPLANE_DB_PASSWORD=你的数据库密码

```

初始化数据库表：

```bash
python3 backend/db.py
```

成功后会输出：

```text

```

当前会创建三张表：

```text
players
scores
rooms_history
```

## 11. Nginx 反向代理

建议使用域名访问，而不是直接暴露 5000。

在宝塔创建网站：

```text
域名：game.example.com
根目录：/www/wwwroot/airplan_game
```

然后设置反向代理：

```text
目标 URL：http://127.0.0.1:5000
发送域名：$host
```

Nginx 配置里需要支持 WebSocket。核心配置如下：

```nginx
location / {
    proxy_pass http://127.0.0.1:5000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_read_timeout 3600s;
    proxy_send_timeout 3600s;
}
```

保存后测试：

```text
http://game.example.com/api/health
```

正常返回：

```json
{"status":"ok"}
```

## 13. 客户端连接公网后端

当前 `code/online_game.py` 支持 `--server` 参数。

例如在玩家电脑上运行：

```bat
D:\anaconda\envs\yl\python.exe code\online_game.py --server https://game.example.com --name playerA
```

另一个玩家运行：

```bat
D:\anaconda\envs\yl\python.exe code\online_game.py --server https://game.example.com --name playerB
```

然后：

```text
玩家 A 创建房间，得到 4 位房间码。
玩家 B 点击加入房间，输入 4 位房间码。

```

如果暂时不用 HTTPS，可以用：

```bat
D:\anaconda\envs\yl\python.exe code\online_game.py --server http://game.example.com --name playerA
```

## 14. 当前主菜单的限制

当前 `code/main_menu.py` 里后端地址仍然写死为：

```python
BACKEND_URL = "http://127.0.0.1:5000"
```

所以如果直接运行主菜单：

```bat
run_game_menu.bat
```

它默认会连接玩家自己电脑的本地后端，而不是阿里云服务器。

部署前建议后续把它改成可配置，例如：

```text
AIRPLANE_SERVER_URL=https://game.example.com
```

或者给 `run_game_menu.bat` 增加服务器地址参数。

在这个改动完成前，公网双人测试优先使用：

```bat
python code\online_game.py --server https://game.example.com --name 玩家昵称
```

## 15. 当前后端的生产限制

当前后端入口是：

```python
socketio.run(app, host="0.0.0.0", port=5000, debug=False, use_reloader=False)
```

并且 SocketIO 当前使用：

```python
async_mode="threading"
```

这适合小规模测试。正式上线前建议优化为：

```text
eventlet 或 gevent 模式
```

并配合稳定的进程守护。否则高并发和长时间运行稳定性不如生产级 WebSocket 服务。

当前房间状态保存在后端内存：

```text
rooms.py 里的 ROOMS
game_engine.py 里的 GAMES
```

因此当前版本只能稳定运行一个后端进程。不要开多个 Python 后端进程做负载均衡，否则不同玩家可能进入不同进程，房间状态不同步。

如果后续要多人规模化，需要增加：

```text
Redis
SocketIO message_queue
房间状态持久化或统一状态服务
```

## 17. 常见问题

### 17.1 浏览器能访问 health，但客户端连不上

检查客户端填写的是不是：

```text
https://game.example.com
```

不要写成：

```text
https://game.example.com/api/health
```

客户端 `--server` 要填根地址。

### 17.2 双人进房后不同步

检查 Nginx 是否配置了：

```nginx
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```

并检查宝塔是否只启动了一个后端进程。

### 17.3 房间创建成功，但另一个人加入失败

检查：

```text
1. 两个客户端是否连接同一个 server 地址
2. 后端是否只有一个进程
3. 房间码是否输错
4. 房间是否超过 15 分钟等待超时
```

### 17.4 排行榜没有数据

检查：

```text
1. MySQL 环境变量是否正确
2. scores 表是否存在
3. 游戏是否正常结算
4. 后端日志是否有数据库错误
```

### 17.5 服务器重启后房间消失

这是当前设计正常现象。

排行榜、玩家、历史房间会保存在 MySQL，但正在等待或正在游戏中的实时房间保存在内存里。服务器重启后，实时房间会丢失。

## 18. 推荐部署顺序

按这个顺序做最稳：

```text
1. 买 ECS，安装宝塔
2. 宝塔安装 Nginx、MySQL、Python 项目管理器
3. 域名 A 记录解析到 ECS
4. 上传项目
5. 创建 MySQL 数据库和用户
6. 初始化数据库表
7. 启动 Flask 后端
8. 测试 /api/health
9. 配置 Nginx 反向代理和 WebSocket
10. 申请 HTTPS
11. 用 online_game.py --server 公网地址测试双人
12. 再改主菜单公网地址配置
13. 最后再打包客户端给玩家
```

## 19. 下一步开发建议

为了让部署更顺手，建议先补三个小功能：

```text
1. 把 main_menu.py 的 BACKEND_URL 改成环境变量或配置文件
2. 增加一个公网客户端启动脚本，例如 run_online_public.bat
3. 增加一份宝塔/生产环境专用 requirements 和启动说明
```

这样部署后，玩家只需要双击启动脚本，不需要自己输入 `--server`。
